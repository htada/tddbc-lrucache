RSpec Code Examples

 

 

LruCache を初期化する場合
は、サイズを渡したら、そのサイズのキャッシュができる.
もし、サイズにマイナス値を渡したら、例外が発生する.
もし、サイズにnilを渡したら、例外が発生する.
もし、サイズに数値以外を渡したら、例外が発生する.
LruCache に値を出し入れする場合
は、キャッシュを入れると、入れたときと同じキーで取りだせる.
は、キャッシュした個数がサイズを越えなければ、キャッシュの中で最も古いキーが取得できる.
は、キャッシュした個数がサイズを越えると、越えた分の値が消えている.
は、つねに、現在キャッシュされている値の個数が取得できる.
もし、キャッシュの中にないキーを取り出すと、nilが返る.
もし、キャッシュが空だったら、最も古いキーとしてnilが返る.
もし、同じキーで別のキャッシュを渡すと、同じキーのキャッシュが上書きされる.
もし、一度最も古いキーをgetしたら、次には古いキーが最も古いキーとして取得できる.
LruCache のキャッシュサイズを変更する場合
は、キャッシュサイズが変更できる.
は、キャッシュサイズを増やすと、キャッシュの内容が変わらない.
は、キャッシュサイズを減らすと、リミットを越えたキャッシュが消える.
は、キャッシュが空ならば、キャッシュサイズを変更しても、例外は発生しない.
もし、新しいキャッシュサイズにマイナス値を渡すと、例外が発生する.
もし、新しいキャッシュサイズにnilを渡すと、例外が発生する.
もし、新しいキャッシュサイズに数値以外を渡すと、例外が発生する.
LruCache のキャッシュの保持期間を変更する場合
は、登録したキャッシュの登録時間が取得できる.
は、getした時、保持期間を過ぎていないキャッシュが消えてはいけない.
は、getした時、保持期間を過ぎたキャッシュがあれば消える.
LruCache が複数スレッドでアグレッシブに実行される場合
は、順番にスレッドを実行すると、通常通り更新と追加ができる.
もし、スレッドを100個ぐらい実行しても、通常通り行進と追加ができる.
もし、主スレッド内でロック中に子スレッドでgetすると、主スレッドがロック解除後に子スレッドの操作がeldest_keyに反映される.
もし、主スレッド内でロック中に子スレッドでputすると、主スレッドがロック解除後に子スレッドのputが反映される.
もし、主スレッド内でロック中に子スレッドでputしつつ主スレッドでもputすると、先に主スレッドのputが反映され、ロック解除後に子スレッドのputが反映される.
もし、スレッド内でロック中に子スレッドでresizeすると、主スレッドがロック解除後に子スレッドのresizeが反映される.
